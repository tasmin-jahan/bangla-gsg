# Hugging Face Export Pipeline: Implementation Plan

We are currently training `BanglaGSG`, a custom 186M parameter hybrid foundation model (GDN + SWA + GQA interleaved).

## Objective
To deploy this model to Hugging Face, we cannot simply upload the `.pt` files because the `transformers` library does not recognize our custom architecture. We need to build an automated export pipeline that wraps our raw model in Hugging Face's `PreTrainedModel` API and utilizes `trust_remote_code=True` via an `auto_map` configuration.

---

## Architecture Requirements

### 1. The Wrapper Classes (To be placed in `src/hf_integration/`)
We need to create two template files that will act as the bridge between Hugging Face's API and our raw PyTorch code.
*   **`configuration_banglagsg.py`**: 
    *   Must subclass `PretrainedConfig`.
    *   Must accept `**kwargs` and pass them to `super().__init__(**kwargs)` to ensure custom fields are properly round-tripped during `save_pretrained` / `from_pretrained`.
*   **`modeling_banglagsg.py`**:
    *   Must subclass `PreTrainedModel`.
    *   Must instantiate our raw `BanglaGSGModel` internally.
    *   Must intercept standard Hugging Face inputs (`input_ids`, `attention_mask`, `labels`).
    *   Must automatically compute Cross-Entropy loss if `labels` are provided, and return a standard `CausalLMOutput`.
    *   **Crucial:** Must implement `get_input_embeddings()` and `get_output_embeddings()` so Hugging Face correctly resolves the tied weights and avoids doubling the safetensors file size.
    *   **Scope:** For v1, explicitly skip implementing `.generate()` / KV-cache routing (`prepare_inputs_for_generation`), as managing hybrid caching for GDN+SWA+GQA is too complex. Raw forward-pass and loss only.

### 2. The Export Script (`scripts/export_to_hf.py`)
This script will be executed once the 5-day training run completes. It must perform the following pipeline:
1.  **Load Checkpoint:** Load the raw `best.pt` file.
2.  **Strip States:** Discard the Muon and AdamW optimizer momentum states to reduce the file size.
3.  **Instantiate Wrapper:** Instantiate the `BanglaGSGForCausalLM` wrapper with the loaded config.
4.  **Load Weights (CRITICAL):** Explicitly load the stripped `state_dict` into the wrapper *after* it is initialized to safely overwrite the random weights generated by `PreTrainedModel.post_init()`.
5.  **Save Weights:** Call `model.save_pretrained(out_dir, safe_serialization=True)` to generate the secure `.safetensors` weights.
5.  **Output Structure Flattening:** Ensure the exported `out_dir` explicitly matches Hugging Face's `trust_remote_code` expectations:
    ```
    output/
    ├── config.json
    ├── modeling_banglagsg.py         # Copied from src/hf_integration/
    ├── configuration_banglagsg.py    # Copied from src/hf_integration/
    ├── model/                        # Copied from src/model/
    │   ├── __init__.py
    │   ├── attention.py
    │   └── ...
    ```
    *(Note: `modeling_banglagsg.py` must use `from .model.model import BanglaGSGModel` to bridge into the subfolder).*
6.  **Inject `auto_map`:** Dynamically update the generated `config.json` to include the `auto_map` dictionary so Hugging Face knows to load our custom files:
    ```json
    "auto_map": {
      "AutoConfig": "configuration_banglagsg.BanglaGSGConfig",
      "AutoModelForCausalLM": "modeling_banglagsg.BanglaGSGForCausalLM"
    }
    ```
7.  **Validation Test:** The script must conclude by spinning up a fresh import loop (simulating an end-user) and successfully loading the model via `AutoModelForCausalLM.from_pretrained(out_dir, trust_remote_code=True)`. It must then explicitly assert:
    - **Weight Tying:** `assert model.model.lm_head.weight.data_ptr() == model.model.embedding.weight.data_ptr()` to guarantee the dual HF/Native tying mechanics didn't silently un-tie the embeddings.
    - **Forward Pass:** Execute a dummy forward pass to verify the export is mathematically sound.
